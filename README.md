# EAgents: 多角色用户模拟库 - 维护指南

欢迎来到 EAgents！这是一个帮助产品经理和设计师**在没有真实用户的场景下，快速获得多视角反馈**的 AI 模拟库。

作为维护人员（用研专家、运营或产品同学），你的主要工作是**丰富角色库**和**补充行业知识**。**你不需要编写任何代码，只需要会写 Markdown 格式的文档即可。**

本文档将手把手教你如何维护这个仓库。

---

## 🎯 你的核心工作区

你只需要关注以下两个文件夹：

1. 📁 **`personas/` (角色画像库)**：存放所有的虚拟角色文件。比如“资深广告投手”、“Z世代女大学生”等。
2. 📁 **`knowledge/` (共享知识库)**：存放行业黑话、产品说明书等。这些知识可以被多个角色共享使用。

---

## 📂 其他文件夹是干什么的？（了解即可）

虽然你不需要修改它们，但了解它们的作用会有所帮助：

- 📁 **`scenarios/` (场景模板)**：存放具体的测试任务要求。比如“如何评审一份 PRD”、“如何评审一张设计稿”。用来约束 AI 的评审行为。
- 📁 **`scripts/` (工程脚本)**：这是给AI用的工具箱，用来把你写的角色文件打包、校验和组装给 AI 引擎。
- 📁 **`tests/` (测试用例)**：用来验证 AI 是否真的扮演好了角色。里面会存放一些标准的问题和满分回答（Golden Data）。
- 📄 **`SKILL.md` (系统说明)**：这是给AI看的全局设计说明书。

> ⚠️ 注意：不要修改 `scripts/` 或根目录下的其他系统文件。

## 📝 任务一：如何创建一个新角色？

当需要让 AI 扮演一种新的用户时，你需要在 `personas/` 目录下创建一个新的 Markdown 文件。

### 第一步：复制模板
在 `personas/` 目录下找到 `_template.md` 文件。**将它复制一份**，并用有意义的名字重命名（比如 `merchant-food-owner.md` 代表餐饮商家老板）。

### 第二步：填写基本信息（Frontmatter）
打开你刚复制的文件，最顶部在 `---` 之间的部分是结构化信息。请像填表一样修改：

```yaml
---
id: merchant-food-owner            # 必须和文件名完全一致（不含 .md）
name: 餐饮商家老板
category: 商家                      # 比如：广告投手 / 商家 / 消费者
sub_category: 本地生活·餐饮         # 比如：服饰 / 餐饮 / 3C
version: 1.0
status: draft                      # 写好并测试没问题后，可以改成 active
imports:                           # 这个角色需要掌握哪些知识？(见下文知识库)
  - knowledge/industry/food-delivery.md
  - knowledge/glossary/merchant-terms.md
---
```

### 第三步：填写 L1-L5 画像内容
模板文件里有详细的注释。你需要用文字把这个角色“写活”：
- **L1 身份层**：TA 是谁？多大？在哪工作？
- **L2 标签层**：TA 的核心痛点是什么？最看重什么？
- **L3 心智层**：TA 遇到问题是怎么思考的？（*这是决定 AI 回答深度的关键*）
- **L4 行为层**：TA 说话是什么口吻？有什么口头禅？遇到特定情况会怎么发脾气？（*这是决定 AI 回答风格的关键*）
- **L5 知识层**：TA 精通什么？半懂不懂什么？什么是 TA 绝对不知道的盲区？

### 第四步：补充 Few-shot（对话示范）
在文件的最后，**必须提供 3-5 段典型的对话片段**。AI 会模仿这些片段的语气来跟你对话。
你可以直接从平时跟真实用户的聊天记录里摘抄几段进来。

## 📚 任务二：如何维护共享知识库？

有时候，多个角色都需要懂同一套规则。比如“广告投手”和“带货主播”都需要懂“千川投放后台”的术语。
我们不需要在每个角色文件里重复写，而是把这些知识写在 `knowledge/` 文件夹里。

### 怎么做？
1. 在 `knowledge/` 下相应的目录（如 `industry/` 或 `glossary/`）新建一个 Markdown 文件，例如 `knowledge/glossary/ad-terms.md`。
2. 在里面直接用大白话解释这些术语是什么意思。
3. **如何让角色学到这些知识？** 回到角色画像文件的顶部（Frontmatter），在 `imports` 列表里把路径加上去就行了。

## ✅ 检查与发布

当你写完了一个新角色或修改了知识库，怎么知道自己写得对不对呢？

如果你懂一点命令行，可以在电脑终端（Terminal）里运行这个命令：
```bash
python3 scripts/lint.py
```
它会像老师改作业一样，告诉你哪个必填项漏填了，或者哪个知识库路径写错了。全绿通过（`PASS`）就代表没问题！

最后，使用常规的 Git 流程提交你的修改即可。底层引擎会自动把你写的这些文档编译成强大的 AI Prompt。

## 💬 任务三：发现问题怎么反馈？

使用过程中如果发现 **角色拟合偏差、知识有误或过时、圆桌讨论效果差、功能建议** 等问题，**请不要直接修改 `personas/` 或 `knowledge/` 文件来"打补丁"**，而是走标准反馈流程，由维护团队评估后统一更新。

### 推荐方式：填写飞书反馈表单（最方便）
直接打开下面的链接，填表提交即可。不需要装任何环境、不需要懂 Git：

👉 **[EAgents 用户反馈表](https://bytedance.larkoffice.com/base/HGLTbRGqdagunpsf5r9cPjobnDd?table=tblEoCPkWbIc3rKk&view=vewriYNmAf)**

表单极度精简，只有两项必填：问题类型、问题描述。可按需提供附件截图。

### 备选方式：让 AI 帮你提交
如果你正在和支持本 Skill 的 AI 对话，直接告诉它"我要反馈一个问题"，AI 会引导你把信息补全后写入同一张多维表格。

### 反馈会去哪儿？
所有反馈会汇总到飞书多维表格 **「EAgents 用户反馈收集」**，维护团队会定期处理并把"已采纳"的内容沉淀回 `personas/` 或 `knowledge/`。
